changelog description :
	
	-Updates to manage web services development
		1-Updating model classes by changing char fields to String, add adding annotation on GegorianCalendar fields
		2-Updating manager classes to ensure the good working of web services
		3-Removing all images from the database to a folder	and updating the old images references in all needed panels.

	-Updates on Laboratory module
		1-Removing the result and material forms in the adding exam process.
		2-Adding more filters on the Laboratory browser :
			-filter exam with all results
			-filter exam with not empty results
			-filter exam with empty results
			-filter exam by patient
			-filter exam by exam type
		3-Adding Button "Manage Exams" on Patients browser witch opens 
	          the Exam browser of the selected patient, showing all his exams and allow 
		  to add new exam or remove existing exam.

	-updates on vaccination module
		1-Adding patient filter on the Patient Vaccines browser
		2-Updating the Clinical Sheet form of a patient by adding the patient vaccines panel of the 
		selected patient.

	-Updates on Accounting module to add a discount system which allows to define many reduction plans
	and to apply one of them to any patient : 
		1-In General Data adding a button to open the reduction plan browser, which shows all the 
		  reduction plans and allows to add a new one, edit or remove an existing.
		2-In the editing panel of a selected patient from the patients browser, adding a field 
		  to set the reduction plan to apply to the selected patient on accounting operations.
		3-Taking in consideration the selected reduction plan on a patient in every accounting operations.

	-In Patient selection dialog, adding a button to add a patient when the user has the privilege.
	to do it. 

	-Adding the possibility to update the medicine stock when Selling.

	-Forcing login at the launching of the application.

Important Note :
	After apply this updates on an open hospital installation, before running the application, be sure to first 
	of all execute that class utility "org.isf.utils.db.ExtractPatientsPhotos".
	That utility has to: 
	 *  extract patient images from the database and store them in the file system. 
	 *  drop the PAT_PHOTO column from patient table.
	 The path to the images folder can be set in the generalData.properties file throw the key "PHOTOSDIR"
